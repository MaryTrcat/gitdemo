# 手语动作评分系统方案

## 推荐实现

核心评分使用本地视觉关键点方案：`Qt Camera -> OpenCV/MediaPipe Hands -> 21个手部关键点 -> GestureScorer -> FeedbackGenerator`。

这样评分逻辑稳定、可解释，适合毕业设计答辩。AI 大模型可作为辅助纠错，不建议作为唯一评分来源。

## 数据流程

1. 摄像头采集实时画面。
2. MediaPipe Hands 识别手部 21 个关键点。
3. 对关键点做归一化，消除手离摄像头远近、画面位置带来的影响。
4. 与标准手势关键点对比，计算位置分、角度分、稳定分。
5. 低分时写入错题本，并生成纠错建议。

## 标准动作来源

管理员上传标准教学视频后，系统可以离线抽取关键帧并保存标准关键点。普通用户登录后直接选择手势练习，不需要重复上传视频。

## 当前代码状态

当前主程序已经禁止用模拟关键点生成正式成绩。后续只需要在 `HandKeypointDetector::detectRealtimeHand()` 中接入真实 MediaPipe/OpenCV 检测结果。

`detectMockHand()` 只保留作开发测试，不再作为正式评分入口。

## AI 增强方向

可增加一个 AI 纠错层，在低分时根据关键点差异生成更自然的提示，例如：

- 手掌偏离标准区域，请将手移动到画面中央。
- 食指弯曲角度不足，请观察标准视频中的指尖方向。
- 动作抖动较大，请保持 1 秒稳定姿态后再评分。
